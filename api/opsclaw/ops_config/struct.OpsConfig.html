Expand description

OpsClaw configuration — wraps the upstream zeroclawlabs Config with SRE-specific fields (projects, notifications, diagnosis, a2a).

Fields§

§inner: Config

The upstream zeroclawlabs configuration.

§targets: Option<Vec<TargetConfig>>

SRE targets (monitored endpoints). Flat form — used when the hierarchy is not yet adopted. Mutually exclusive with projects.

§projects: Vec<ProjectConfig>

Hierarchical form: products wrapping environments wrapping targets. Mutually exclusive with flat targets — see ADR-005.

§notifications: Option<OpsClawNotificationConfig>

Notification delivery settings for alerts.

§diagnosis: DiagnosisConfig

LLM diagnosis configuration.

§a2a: Option<A2aConfig>

A2A (Agent-to-Agent) protocol configuration.

§postgres: Option<Vec<PostgresInstanceConfig>>

Postgres instances for the postgres tool (driver-based). Kept at the root because the driver-based pool doesn’t yet have an equivalent slot in EndpointsConfig.

pub async fn decrypt_secret(&self, value: &str) -> Result<String>

Resolve a secret reference from config to its plaintext value.

Accepts any of: enc2:<hex> (encrypted store), enc:<hex> (legacy XOR, auto-upgraded on next save), env:<NAME> (process env var), k8s:<ns>/<name>/<key> (mounted Secret volume, kube API fallback), or a bare plaintext value (returned as-is for backward compat).

Async because the k8s mounted-file path does real I/O; the other backends are cheap.

Source

pub async fn save(&self) -> Result<()>

Serialize the full OpsConfig (including opsclaw-specific fields) to self.config_path. Must be called instead of Config::save() whenever the caller may have modified projects, notifications, diagnosis, or a2a, because Config::save() only serializes the inner zeroclaw fields and would silently drop those fields.

pub fn resolve_target(&self, address: &str) -> Result<ResolvedTarget<'_>>

Resolve an address (project::environment::target, or short forms when unambiguous) to a ResolvedTarget. Works for both the flat targets list and the hierarchical projects form.

Short addresses:

  • target — permitted when there is only one project and one environment, or the flat form is in use.
  • environment::target — permitted when there is only one project.

pub fn validate_hierarchy(&self) -> Result<()>

Validate hierarchy invariants:

  • Flat [[targets]] and hierarchical [[projects]] are mutually exclusive.
  • Project names are unique.
  • Environment names are unique within a project.
